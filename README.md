## 1. Project Overview

**The Greek Directory (TGD)** is a United StatesŌĆōfocused directory of Greek-owned businesses. The live domain is `https://thegreekdirectory.org`. The site is a static-first PWA (no framework, vanilla JS + Tailwind CDN) hosted on **Cloudflare Pages**, backed by **Supabase** (Postgres 17, us-west-2) for all dynamic data.

The business model has four listing tiers: **Free ŌåÆ Verified ŌåÆ Featured ŌåÆ Premium**. Businesses can claim their listing, sign in to a business dashboard, and edit their profile. Admins manage all listings through a separate admin portal authenticated via GitHub PAT.

### 4.3 RLS Summary

- **All `auth.jwt()` / `auth.uid()` / `auth.role()` calls in policies are wrapped in `SELECT (...)`** for per-row performance.
- **Admin** is identified by `(SELECT auth.jwt()) -> 'app_metadata' ->> 'role' = 'admin'` via `analytics_is_admin()`.
- **`listings`**: anon SELECT (visible only), owner UPDATE (own listing via `business_owners` join), admin ALL.
- **`business_owners`**: owner SELECT/UPDATE (own rows), admin ALL.
- **`category_subcategories`**: anon SELECT, service role INSERT/UPDATE.
- **`analytics_events`**: approved-path INSERT only (no unconditional open insert).
- **`listing_requests`**: public INSERT with field validation (`WITH CHECK` on required fields).

### 4.4 Extensions

- `pg_trgm` ŌĆö relocated to `extensions` schema. DB-level `search_path` includes `extensions`. Functions calling `similarity()` have `SET search_path = public, extensions`.

### 4.5 Indexes

- Unique slug: single unique constraint on `listings.slug` (duplicates were cleaned up).
- FK covering indexes exist on `analytics_sessions(listing_id)` and `analytics_listing_daily(listing_id)`.

## 5. Authentication Model

### Admin Authentication
- Admin portal (`admin.html`) is authenticated via **GitHub Personal Access Token**.
- The token is passed as `x-github-token` header to the `admin-proxy` Supabase Edge Function.
- `admin-proxy` validates the token server-side and uses `SUPABASE_SERVICE_ROLE_KEY` for all DB operations.
- Admin Supabase users have `app_metadata.role = 'admin'` set via `||` jsonb merge (never overwrites other metadata).
- Token is stored in `localStorage` as `tgd_admin_token`.

### Business Owner Authentication
- Email + password signup via `supabaseClient.auth.signUp()`.
- Magic link fallback via `send-magic-link` Edge Function (admin-triggered).
- Password reset via `send-password-reset` Edge Function.
- On signup, `business_owners.confirmation_key` is validated, then nulled and `owner_user_id` is set.
- `owner_user_id` format: `{emailUsernameAlphanumeric}{listingId}`.
- Auth state is managed through `window.TGDAuth` namespace (exported from `js/supabase-config.js`).
- Business dashboard is at `business.html`. Redirect target for magic links.

### Supabase Client

### `adminProxy(action, payload)` ŌĆö Central Admin API Call
```js
async function adminProxy(action, payload = {}) {
  const res = await fetch(
    'https://luetekzqrrgdxtopzvqw.supabase.co/functions/v1/admin-proxy',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'x-github-token': adminGithubToken,
      },
      body: JSON.stringify({ action, payload }),
    }
  );
  const json = await res.json();
  if (!json.success) throw new Error(json.error || 'Admin proxy error');
  return json.data;
}
```

## 10. Listing Pages & Static Generation

### Template: `listing-template.html`
A static HTML template with `{{PLACEHOLDER}}` tokens that get replaced at generation time.

### Generation Flow (Admin ŌåÆ "Generate Page" button)
1. Admin clicks ­¤ö© on a listing row.
2. `generateListingPage(listingId)` is called.
3. Fetches `listing-template.html` from GitHub raw URL.
4. Calls `generateTemplateReplacements(listing)` and `generateTemplateReplacementsPart2(listing)` to build a replacements map.
5. Replaces all `{{KEY}}` tokens.
6. Calls `saveToGitHub('listing/{slug}.html', content, businessName)` which uses the GitHub Contents API with the stored PAT.
7. Calls `updateSitemap()` to regenerate `sitemap.xml`.

### Static File Path
```
/listing/{slug}.html   ŌåÆ   https://thegreekdirectory.org/listing/{slug}
```

Cloudflare Pages handles `.html` extension stripping.

## 18. Critical Patterns & Rules

### DO NOT break these patterns:

**1. `apply_migration` for DDL, `execute_sql` for one-off DML**
Migrations are transactional, logged, and the reliable path for schema + RLS changes.

**2. Column type changes on tables with RLS policies**
Must follow: drop dependent policies ŌåÆ alter column ŌåÆ recreate policies. Direct `ALTER COLUMN TYPE` will fail if policies reference the column.

**3. `int4 ŌåÆ bigint` conversions**
Requires shadow-column pattern: add bigint column ŌåÆ backfill ŌåÆ SET NOT NULL ŌåÆ drop old ŌåÆ rename. Direct alteration fails with dependency errors.

**4. `pg_trgm` + `search_path`**
After moving `pg_trgm` to `extensions` schema: drop/recreate GIN index, update DB-level `search_path` to include `extensions`, add `SET search_path = public, extensions` to any function calling `similarity()`. Note: `similarity()` returns `double precision`; functions with `real` return type must cast with `::real`.

**5. jsonb upsert pattern**
Use `ON CONFLICT (category) DO UPDATE SET col = EXCLUDED.col`. For populating jsonb columns, one `execute_sql` UPDATE per row with inline `'{...}'::jsonb` cast is more reliable than batch statements.

**6. `||` for auth metadata**
When setting `app_metadata.role`, always use `||` jsonb merge operator to avoid overwriting existing metadata fields.

**7. Admin check in RLS = `analytics_is_admin()`**
All policies use this function consistently. Do not inline the JWT check.

**8. Never guess GitHub file contents from raw URLs**
`web_fetch` on guessed GitHub raw URLs returns nothing. Reconstruct data from the DB instead.

**9. Subcategories are always loaded from DB**
Never hardcode subcategory lists in new features. Always query `category_subcategories` table.

**10. `places_url_ending` is `text[]`**
Was converted from `text`. Supports multiple local page slugs per listing. Default is empty array `{}`.
